-------- ATTRIBUTES --------

Syntax:
	[x?(x...), ...?]
	<expression>

List:

	"[vertex]"
		Valid: Function
		Requires: None,
		Description: Mark a function as the vertex shader entrypoint

	"[pixel]"
		Valid: Function
		Requires: None,
		Description: Mark a function as the pixel shader entrypoint

	"[constants]"
		Valid: ConstantBuffer<>
		Requires: None,
		Description: Indicate that a constant buffer should use 32bit constants instead of a buffer object.
					 Note: Shaders are limited to 128 bytes of data.

	"[ialayout(Name: string, Offset: int?, Slot: int?, Class: InputClassification?, Format: InputElementFormat?)]"
		Valid: Function
		Requires: "vertex"
		Arguments:
			Name: Name of the input element to alter.
			Offset: The new byte offset to use for the input element.
			Slot: The new input slot to use for the input element
			Class: The input classification to use for the input element
			Format: The new format to use for the input element
		Description: Change the data of an input element.

	"[bindgroup(Group: string)]"
		Valid: StructuredBuffer<>, ConstantBuffer<>, Texture1..3D<>
		Requires: None,
		Arguments:
			Group: Name of the bind group to associate this resource with
		Description: Set a custom bind group to associate the resource with other then the default.

	"[property(Name: string?, Default: PropertyDefault?)]"
		Valid: Texture1..3D<>, TextureCube

	"[display(Display: PropertyDisplay)]"
		Valid: All property capable types
		Requires: Propety type
		Arguments:
			Display: The display to use for this property
		Description: Change the a certain property is displayed in the inspector.

	"[global(Name: string?)]"
		Valid: StructuredBuffer<>, ConstantBuffer<>, Texture1..3D<>, TextureCube
		Requires: None,
		Arguments:
			Name: Global name to use instead of decleration name.
		Description: Makes the variable bindable with globals instead of parameters.
